# 前端更新问题排查指南

## 问题：前端已构建但页面没有变化

### 可能的原因和解决方案

#### 1. 浏览器缓存问题（最常见）

**解决方案：**
- **强制刷新页面**：
  - Windows: `Ctrl + F5` 或 `Ctrl + Shift + R`
  - Mac: `Cmd + Shift + R`
- **清除浏览器缓存**：
  - Chrome/Edge: 按 `F12` 打开开发者工具 → 右键刷新按钮 → 选择"清空缓存并硬性重新加载"
  - 或者：设置 → 隐私和安全 → 清除浏览数据 → 选择"缓存的图片和文件"

#### 2. 后端服务未重启

**解决方案：**
1. 停止当前运行的后端服务（按 `Ctrl + C`）
2. 重新启动 `app.py`
3. 确保看到日志中显示：`Serving frontend from ...`

#### 3. 检查构建文件是否正确

**验证步骤：**
```powershell
# 检查 dist 目录是否存在
Test-Path "src\handlers\client\rtc_client\frontend\dist\index.html"

# 查看 dist 目录内容
dir "src\handlers\client\rtc_client\frontend\dist"
```

**应该看到：**
- `index.html`
- `assets/` 目录（包含各种 .js 和 .css 文件）

#### 4. 检查访问的URL

**正确的访问地址：**
- 视频聊天：`https://127.0.0.1:8282/ui/` 或 `https://127.0.0.1:8282/ui/videochat`
- 控制台：`https://127.0.0.1:8282/ui/console`

**注意：**
- 使用 `https://` 而不是 `http://`
- 端口是 `8282`
- 路径是 `/ui/` 开头

#### 5. 检查后端日志

启动后端后，应该看到类似这样的日志：
```
Serving frontend from F:\BaiduNetdiskDownload\OpenAvatarChat-250916\OpenAvatarChat\src\handlers\client\rtc_client\frontend\dist
```

如果没有看到这条日志，说明静态文件服务没有正确设置。

#### 6. 验证前端文件是否最新

**检查方法：**
1. 打开浏览器开发者工具（F12）
2. 切换到"网络"（Network）标签
3. 刷新页面
4. 查看加载的 JS/CSS 文件的时间戳
5. 确认文件是刚刚构建的

#### 7. 检查控制台错误

打开浏览器开发者工具（F12），查看"控制台"（Console）标签，看是否有错误信息。

常见错误：
- `404 Not Found` - 文件路径不对
- `Failed to load resource` - 静态文件服务未正确配置
- `CORS error` - 跨域问题（通常不会出现）

## 快速解决步骤

1. **停止后端服务**（如果正在运行）
2. **确认前端已构建**：
   ```powershell
   cd src\handlers\client\rtc_client\frontend
   npm run build
   ```
3. **重启后端服务**：
   ```powershell
   cd F:\BaiduNetdiskDownload\OpenAvatarChat-250916\OpenAvatarChat
   python app.py
   ```
4. **清除浏览器缓存并强制刷新**：
   - 按 `Ctrl + Shift + Delete` 清除缓存
   - 或按 `Ctrl + F5` 强制刷新
5. **访问正确的URL**：
   - `https://127.0.0.1:8282/ui/console`

## 如果还是不行

1. **检查 dist 目录的修改时间**：
   ```powershell
   (Get-Item "src\handlers\client\rtc_client\frontend\dist\index.html").LastWriteTime
   ```
   确认是最新的构建时间

2. **检查后端日志**，确认静态文件服务已正确挂载

3. **尝试访问静态文件直接URL**：
   - `https://127.0.0.1:8282/ui/index.html`
   - `https://127.0.0.1:8282/ui/assets/index.js`

4. **检查浏览器控制台**，查看具体错误信息
