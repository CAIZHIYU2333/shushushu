# 前端强制刷新指南

## 问题：前端已构建但页面没有更新

### 快速解决方案

#### 1. 清除浏览器缓存（最重要！）

**Chrome/Edge 浏览器：**
1. 按 `F12` 打开开发者工具
2. **右键点击浏览器地址栏的刷新按钮**
3. 选择 **"清空缓存并硬性重新加载"**（Empty Cache and Hard Reload）

**或者：**
1. 按 `Ctrl + Shift + Delete`
2. 选择"缓存的图片和文件"
3. 时间范围选择"全部时间"
4. 点击"清除数据"

**或者使用快捷键：**
- `Ctrl + F5` - 强制刷新
- `Ctrl + Shift + R` - 硬性重新加载

#### 2. 确认后端服务已重启

**重要：** 修改代码后必须重启后端服务！

1. 停止当前运行的服务（按 `Ctrl + C`）
2. 重新运行 `app.py`
3. 查看日志，确认看到：`Serving frontend from ...`

#### 3. 检查构建文件时间

确认前端文件是最新构建的：

```powershell
# 查看构建文件的时间
(Get-Item "src\handlers\client\rtc_client\frontend\dist\index.html").LastWriteTime
(Get-Item "src\handlers\client\rtc_client\frontend\dist\assets\index.js").LastWriteTime
```

如果时间不是刚刚构建的，需要重新构建：

```powershell
cd src\handlers\client\rtc_client\frontend
npm run build
```

#### 4. 使用无痕模式测试

打开浏览器的无痕/隐私模式，访问：
- `https://127.0.0.1:8282/ui/console`

这样可以排除浏览器缓存的影响。

#### 5. 检查浏览器控制台

按 `F12` 打开开发者工具，查看：
- **控制台（Console）** - 看是否有错误
- **网络（Network）** - 查看加载的文件，确认文件是最新的

在 Network 标签中：
1. 勾选 "Disable cache"（禁用缓存）
2. 刷新页面
3. 查看加载的 JS/CSS 文件的时间戳

#### 6. 验证静态文件服务

在浏览器中直接访问静态文件，确认能正常加载：
- `https://127.0.0.1:8282/ui/index.html`
- `https://127.0.0.1:8282/ui/assets/index.js`

如果这些文件能正常加载，说明静态文件服务正常。

## 常见问题

### Q: 为什么清除缓存后还是旧页面？

A: 可能的原因：
1. 后端服务没有重启
2. 构建文件不是最新的
3. 浏览器还在使用 Service Worker 缓存

**解决方法：**
- 重启后端服务
- 重新构建前端
- 在开发者工具中，Application 标签 → Service Workers → Unregister

### Q: 如何确认前端文件是最新的？

A: 检查文件修改时间：
```powershell
Get-ChildItem "src\handlers\client\rtc_client\frontend\dist" -Recurse | 
    Select-Object FullName, LastWriteTime | 
    Sort-Object LastWriteTime -Descending | 
    Select-Object -First 5
```

### Q: 构建后需要做什么？

A: 
1. **必须重启后端服务**
2. 清除浏览器缓存
3. 强制刷新页面（Ctrl + F5）
