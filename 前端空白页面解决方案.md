## 现在请按以下步骤操作

### 1. 重启后端服务（必须！）

**重要：** 修改代码后必须重启后端服务！

1. 停止当前运行的服务（按 `Ctrl + C`）
2. 重新运行 `python app.py`
3. 查看日志，确认看到：`Serving frontend from ...`

### 2. 清除浏览器缓存

在浏览器中：
1. 按 `F12` 打开开发者工具
2. **右键点击地址栏的刷新按钮**
3. 选择 **"清空缓存并硬性重新加载"**（Empty Cache and Hard Reload）

或者：
- 按 `Ctrl + Shift + Delete`
- 选择"缓存的图片和文件"
- 时间范围选择"全部时间"
- 点击"清除数据"

### 3. 访问正确的 URL

**重要：** 不要直接访问 `/ui/index.html`！

请访问以下 URL 之一：

- **视频聊天页面：** `https://127.0.0.1:8282/ui/` 或 `https://127.0.0.1:8282/ui/videochat`
- **控制台页面：** `https://127.0.0.1:8282/ui/console`

**注意：**
- 使用 `https://` 而不是 `http://`
- 端口是 `8282`
- 路径是 `/ui/` 开头，但**不要**加 `index.html`

### 4. 如果还是空白，检查浏览器控制台

1. 按 `F12` 打开开发者工具
2. 切换到 **Console（控制台）** 标签
3. 查看是否有红色错误信息
4. 切换到 **Network（网络）** 标签
5. 勾选 **"Disable cache"（禁用缓存）**
6. 刷新页面
7. 查看加载的文件，确认：
   - `index.js` 状态是 `200 OK`
   - `index.css` 状态是 `200 OK`
   - 没有 `404` 或 `500` 错误

### 5. 使用无痕模式测试

打开浏览器的无痕/隐私模式，访问：
- `https://127.0.0.1:8282/ui/console`

这样可以排除浏览器缓存和扩展程序的影响。

## 常见问题排查

### Q: 为什么不能直接访问 `/ui/index.html`？

A: 虽然静态文件服务可以提供 `index.html`，但 Vue Router 的 base 路径是 `/ui/`，直接访问 `/ui/index.html` 会导致路由无法正确匹配。应该访问 `/ui/` 或 `/ui/videochat` 等路由路径。

### Q: 页面还是空白怎么办？

A: 按以下顺序检查：

1. **确认后端服务已重启** - 这是最常见的遗漏
2. **清除浏览器缓存** - 使用硬性重新加载（Ctrl + Shift + R）
3. **检查浏览器控制台** - 查看是否有 JavaScript 错误
4. **检查网络请求** - 确认所有文件都正常加载（状态码 200）
5. **使用无痕模式** - 排除缓存和扩展程序影响

### Q: 控制台显示对象 `{ avatar_config: {}, rtc_configuration: null }` 是什么意思？

A: 这是正常的！这是前端初始化时从后端获取配置信息的日志输出。如果看到这个，说明：
- ✅ JavaScript 代码正在执行
- ✅ 前端应用已加载
- ✅ 后端 API 连接正常

如果页面还是空白，可能是：
- 路由没有匹配到组件
- 组件加载失败（检查控制台错误）
- CSS 问题导致内容不可见

### Q: 如何确认前端文件是最新的？

A: 检查文件修改时间：
```powershell
(Get-Item "src\handlers\client\rtc_client\frontend\dist\index.html").LastWriteTime
```

如果时间不是刚刚构建的，需要重新构建：
```powershell
cd src\handlers\client\rtc_client\frontend
npm run build
```
